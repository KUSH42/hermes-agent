Gotchas hit during the hermes TUI migration and roadmap spec review (Textual 8.2.3):

- **`work` import path:** `from textual import work`, NOT `from textual.work import work` — the latter does not exist.
- **`RichLog` line counting:** Use `len(log.lines)`, not `log.line_count` — no such attribute.
- **`get_css_variables()` called during `super().__init__()`:** Any instance attrs it reads won't exist yet. Use `getattr(self, "_skin_vars", {})`.
- **pytest with `-n auto` in addopts:** Textual `app.run_test()` needs serial execution. Override with `pytest -o "addopts="` for TUI tests. Fix properly: add `--ignore=tests/tui/` to pyproject.toml addopts.
- **`RichLog.write()` has no `markup` kwarg** — set at construction; pass `Text.from_markup(...)` for styled lines.
- **`widget.styles.animate()` does not exist.** Use CSS `transition:` declarations + direct property mutation (`widget.styles.opacity = 1.0`).
- **`call_later` takes no delay.** `App.call_later(callback, *args)` — first arg IS the callback. Use `set_timer(delay_seconds, callback)` for delayed execution.
- **`display: none` blocks CSS transitions.** Set `display = True` first, then animate. Hide: animate → `set_timer(duration + 0.016, set_display_false)`.
- **`Markdown.update()` requires DOM mount.** Calling it on an unmounted widget raises `NoApp`. Order: mount first, then update.
- **`Markdown.BLOCKS` for fence substitution.** To use a custom `MarkdownFence` subclass: `class MyMarkdown(Markdown): BLOCKS = {**Markdown.BLOCKS, "fence": MyFence}`. The `parser_factory` param configures the tokenizer, NOT widget dispatch.
- **Unsafe ctrl-key bindings:** `ctrl+e` (cursor_line_end), `ctrl+f` (delete_right_word), `ctrl+j` (newline≡Enter), `ctrl+k` (delete_to_end), `ctrl+p` (cursor_up/history), `ctrl+c` (app quit). Use F-keys instead.
- **`align: center middle` aligns children, not self.** For modal centering: use `ModalScreen` and set `align: center middle` on the Screen, not the dialog container.
- **Child widget cannot watch parent reactive.** Add `watch_<reactive>` on the owner, call `self.query_one(Child).refresh()`.
- **CSS property names in `DEFAULT_CSS` strings use kebab-case** (`max-height`), not snake_case (`max_height`). Python attrs use snake_case.
- **`easing` name is `"in_out_cubic"`** (not `"ease_in_out_cubic"` — KeyError at runtime).
- **`styles.animate("opacity", ...)` causes test flakiness in full suites.** The 60fps `set_interval` callbacks compete with `RichLog._deferred_renders` commit ticks. Tests that pass in isolation fail after 200+ tests. Fix: skip under `PYTEST_CURRENT_TEST` env var. Tests asserting final opacity==1.0 pass trivially without animation.
- **`styles.opacity = 0.0` in `on_mount` prevents child layout.** Textual skips sizing children with opacity=0, so `RichLog._deferred_renders` never commit. Always set opacity=0 AFTER first render (in `call_after_refresh` callback), not in `on_mount` directly.
- **`watch_browse_mode` resets itself to False when no ToolHeaders exist.** Setting `app.browse_mode = True` in a test with no tool blocks mounted will immediately be reset by the watcher (`if not list(self.query(ToolHeader)): self.browse_mode = False`). Tests that rely on browse mode must first mount at least one ToolHeader/ToolBlock.
- **Input spinner text: use `placeholder`, not a sibling overlay.** Setting `inp.placeholder = spinner_text` while the input is disabled (agent running) is zero-reflow. Toggling `inp.display = False/True` or a sibling overlay causes a layout reflow every tick and breaks spatial anchoring.
- **`Widget.mount(child, before=other)` resolves parent from `other.parent`, not `self`.** `panel.mount(x, before=output.live_line)` silently mounts `x` into `output` (OutputPanel), not `panel` (MessagePanel). Always use an anchor that is a direct child of the intended parent container.
- **`TEXTUAL_FPS=200` saturates the event loop.** Setting env var to 200 drives Textual's render loop AND any `set_interval(1/MAX_FPS, ...)` probes at 200 Hz — 400+ callbacks/second. Causes black areas, render corruption, scroll glitches. Default of 60 is sufficient for a chat TUI.
- **`display:none → display:block` on a widget at the wrong DOM position causes scroll jumps.** ThinkingWidget and ToolPendingLine toggling at the top of OutputPanel (above all messages) shifted virtual height and corrupted scroll position every activate/deactivate cycle. Keep toggling widgets adjacent to their logical position in the layout (e.g., just above LiveLineWidget via compose order).
- **`RichLog.write(expand=True)` collapses to narrow width when called before layout.** `expand=True` computes `max(scrollable_content_region.width, 0)`. Before the first layout pass, `scrollable_content_region.width = 0` → every write wraps at ~0 chars. Fix: in the subclass `write()` override, check `if scrollable_content_region.width < 4: use app.size.width` as fallback, then pass `width=` explicitly with `expand=False, shrink=False`.
- **`RichLog.write()` defaults to `shrink=True, expand=False`, using `app.console.width` as the cap.** `app.console` is created at `App.__init__` time with the terminal size at startup — it never updates on resize. If the user starts narrow and expands, all subsequent writes wrap at the startup width. Fix: override `write()` in a subclass to default `expand=True`, which uses `scrollable_content_region.width` (live widget width) when larger.
- **`Static` subclasses: pass initial content to `__init__`, not `on_mount`.** `on_mount` is dispatched as an async message — the widget renders before it fires. Any content set via `self.update()` in `on_mount` is invisible on the first paint. Pass the renderable directly to `super().__init__(content, ...)` instead.
- **`Widget.mount()` must be awaited when children must be present before `add_class("--visible")`.** Unawaited `self.mount(*children)` schedules the DOM change; the widget renders before children are attached. Await the result to guarantee children exist at first paint.
- **`position: absolute` with `width/height: auto` does not size from children.** Absolutely-positioned containers must have explicit `styles.width` and `styles.height` set at runtime; `auto` resolves to zero under absolute layout. Compute dimensions from content and assign via `self.styles.width = N`.
- **`Widget.remove()` is async — await it when ordering matters.** Unawaited `child.remove()` schedules the removal; stale children remain in the DOM when the next `mount()` runs. In re-show patterns (`show()` teardown then rebuild), await each `child.remove()` first.
- **`call_from_thread` is App-only in Textual 8.x.** `Widget` does not have `call_from_thread`. Inside a `@work(thread=True)` method on a widget, use `self.app.call_from_thread(fn)` — NOT `self.call_from_thread(fn)`. The latter raises `AttributeError` at runtime.
- **`render_halfblock` / PIL RGBA→RGB composites against white.** `image.convert("RGB")` fills transparent pixels with white; transparent-background images (e.g. matplotlib math PNGs) appear as solid white rectangles. Fix: composite RGBA against `#0d0d0d` before conversion: `bg = PILImage.new("RGB", rgba.size, (13, 13, 13)); bg.paste(rgba, mask=rgba.split()[3])`.
- **`_split_row` splits on all `|` including inside backtick spans.** The GFM table row splitter (`agent/rich_output.py`) originally did `s.split("|")` which splits on pipes inside `` `code | here` `` and `\|` escapes. Fixed to scan with backtick-fence tracking.
- **`EnterWorktree` creates branch from repo HEAD, not current branch.** If HEAD diverges from your active feature branch, `EnterWorktree` creates the worktree from the wrong base. Workaround: `git worktree add .claude/worktrees/NAME -b BRANCH BASE_BRANCH` manually, then use `EnterWorktree path=...` to enter it.
- **CSS type selectors match `_css_type_name`, NOT base-class names.** `"MessagePanel RichLog { overflow-y: hidden }"` does NOT match `CopyableRichLog` (whose `_css_type_name` is `"CopyableRichLog"`). Subclasses must either (a) declare their own `DEFAULT_CSS`, or (b) be listed explicitly in parent CSS rules. This silently lets inherited `overflow-y: scroll` from `RichLog` stay active on `CopyableRichLog`, making `allow_vertical_scroll=True` → `_on_mouse_scroll_up` stops the event → `OutputPanel` never receives scroll.
- **App-level `on_click(button=1)` focus-stealing breaks OutputPanel select/scroll.** Focusing `#input-area` on every left click steals focus from the output panel, cancelling text selection and disrupting scroll state. Guard the focus-steal by walking `event.widget`'s parent chain and returning early if any ancestor is `OutputPanel`.
- **Hidden focused widget swallows scroll events.** Calling `remove_class("--visible")` (→ `display:none`) on a focused widget does not always trigger `on_blur` in Textual 8.x. The hidden widget retains focus and swallows subsequent mouse-scroll events. Fix: explicitly call `self.app.query_one("#input-area").focus()` in `dismiss()` when `self.app.focused is self`.
- **`action_dismiss()` must cancel any pending timers/debounces before hiding.** If a widget schedules a deferred callback (e.g. `_debounce_handle = self.set_timer(0.15, fn)`) and the user dismisses before it fires, the timer fires against the now-hidden widget and mutates the DOM (remove/mount children). Pattern: store the handle and call `handle.stop(); handle = None` at the top of `action_dismiss()`. Applied to `HistorySearchOverlay` debounce (2026-04-12).
- **Scrollbar gutters render even when `scrollbar-size-vertical: 0`.** The gutter column (right edge) and corner pixel (bottom-right) still paint with `scrollbar-background` / `scrollbar-corner-color`, which default to Textual theme colors darker than `$app-bg`. Visible as a dark strip on the right edge and a dark pixel at the bottom. Fix: `* { scrollbar-background: $app-bg; scrollbar-background-hover: $app-bg; scrollbar-background-active: $app-bg; scrollbar-corner-color: $app-bg; }` in `hermes.tcss`.
- **`App { background: X }` does NOT set the background for transparent child widgets.** Transparent widgets inherit from the **Screen** widget, not from HermesApp. `Screen.DEFAULT_CSS` sets `background: $background` (#121212). Without overriding `Screen { background: $app-bg }` in `hermes.tcss`, transparent widgets (HintBar, TitledRule, margin areas) show `#121212` while explicitly-backgrounded widgets (CopyableRichLog, #input-row) show `#1E1E1E` — visible two-tone artifact. Fix: always set **both** `Screen { background: $app-bg }` AND `HermesApp { background: $app-bg }`.
- **`PlainRule` reads colors once at `__init__` — no hot-reload.** `PlainRule.__init__` calls `_skin_color("rule_end", "#2A2A2A")` at construction time. Changes to `rule-bg-color` via skin hot-reload are not reflected until restart. Fix: call `self.app.get_css_variables()` inside `render()` on each paint. `TitledRule` already uses `_live_colors()` for this; `PlainRule` was updated 2026-04-12 to match.
- **`Text.stylize(style, start, end)` requires a `Style` object, not a bare hex string.** `result.stylize("#FF0000", 0, 3)` is accepted by Rich's parser but is fragile across versions. The correct call is `result.stylize(Style(color="#FF0000"), start=i, end=i+1)` — overrides only the foreground color while preserving bold/dim/italic on existing spans. Bare hex strings may silently fail or produce unexpected merges in Rich ≥15.
- **`self.content_size.width` vs `self.size.width` for padding-aware measurement.** When a widget has `padding: 0 1` (1 char each side), `self.size.width` is the full outer width (including padding). `self.content_size.width` returns the inner width after padding is subtracted. Use `content_size.width` when computing how many characters of content fit — e.g. for width-responsive hint string selection. Using `size.width` causes text to overflow by 2 chars at standard 80-col terminal width.
- **Nested `overflow-y: auto` containers corrupt dirty-rect renders after scroll.** A `RichLog` with `overflow-y: auto` inside a `ScrollableContainer` accumulates internal scroll offset (e.g. from streaming `scroll_end` calls). After the parent container scrolls to a different position, mouse movement triggers dirty-rect recalculation using the stale internal offset → wrong lines render (visual corruption). Fix: use `overflow-y: hidden` on all inner content widgets; keep exactly ONE scroll container in the widget tree (WhatsApp model). Applied to `ToolBodyContainer CopyableRichLog` in 2026-04-12.
- **`$surface` = `$app-bg` (#1E1E1E) in Textual's default dark theme.** `$panel` = #242F38 (blue-tinted). To create a neutral elevated background for completion overlays or floating panels, use `$surface-lighten-1` (#2D2D2D) — NOT `$surface` (zero elevation) and NOT `$panel` (blue tint). `$surface-lighten-N` tokens are guaranteed brighter-than-base by Textual's own color system. Verified 2026-04-12 via `app.get_css_variables()["surface"]`.
- **`RichLog` / `ScrollView` subclasses: use `can_focus=False` to prevent scroll-to-top. Text selection works via `ALLOW_SELECT`, not `can_focus`.** In Textual 8.x, `Screen.set_focus()` does NOT call `widget.scroll_visible()` — it calls `self.call_later(scroll_to_center, widget)` which walks up via `scroll_to_widget` → `container.scroll_to_region()`. A `scroll_visible()` no-op on the widget is therefore completely ineffective. **Correct fix: `can_focus=False`** — prevents the focus event entirely, preventing the `scroll_to_center` chain. Text selection is independent: Textual 8.x gates it on `ALLOW_SELECT` (class attribute, default `True` on `Widget` and `RichLog`). Mouse selection events (MouseDown/Move/Up) are delivered by cursor position, not focus, so drag-to-select works with `can_focus=False`.
- **`compose()` runs asynchronously — `compose()` children have no `.parent` until the compose pass completes.** Calling `self.mount(child, after=self._header)` from a method that may fire on the same event-loop tick as the widget's DOM mount (e.g., `complete()` called from `ResponseFlowEngine` when a fence opens and closes quickly) raises `MountError: Unable to find relative location because it has no parent`. Root cause: `compose()` is dispatched as a deferred message; the widget is in the DOM but `self._header.parent is None` until that message is processed. **Fix:** pre-create all children in `__init__`, yield them from `compose()` normally, and use CSS class toggling (`add_class("--complete")` → `display: none → block`) instead of dynamic `mount()` calls.
- **`COMPONENT_VAR_DEFAULTS` (ThemeManager) must include ALL custom vars read via `get_css_variables()` in Python.** TCSS `$var-name` declarations are **compile-time substitutions** — the parser expands them when the stylesheet is parsed. They are NOT injected into the runtime CSS-variables dict returned by `App.get_css_variables()`. If Python code calls `v = self.app.get_css_variables(); v.get("primary-darken-3", fallback)`, the key will always miss unless `"primary-darken-3"` is explicitly added to `COMPONENT_VAR_DEFAULTS`. Symptom: `_live_colors()` always hits the fallback color (often near-invisible). Fix: every `$var-name` referenced in Python `get_css_variables()` lookups must also exist in `COMPONENT_VAR_DEFAULTS`.
- **`flush_live()` commits a pending buffered line AFTER the last chunk's `scroll_end` was queued.** `StreamingBlockBuffer` holds one line in lookahead for setext-heading detection. The sentinel (None) path calls `flush_live()` which commits this buffered line to `response_log`, extending `max_scroll_y` — but the `scroll_end` queued for the last chunk already fired before this write. Result: last line scrolls off the bottom after streaming ends. Fix: queue an additional `call_after_refresh(panel.scroll_end)` in the sentinel path, after `flush_live()`.
- **User message echo should always scroll, regardless of `_user_scrolled_up`.** `echo_user_message()` was guarded by `if not panel._user_scrolled_up`. If the user had scrolled up and then submitted, their own message mounted at the bottom but the panel stayed at the old scroll position — user couldn't see what they sent. Submission implies intent to see the exchange: unconditionally reset `_user_scrolled_up = False` and call `scroll_end` in `echo_user_message()`.
- **`scroll_relative(animate=False)` defers the scroll via `call_after_refresh` — the screen does NOT update until the next refresh tick.** `scroll_to()` (called by `scroll_relative`) schedules the actual position change via `self.call_after_refresh(self._scroll_to, ...)` unless `immediate=True` is passed. If nothing else triggers a refresh (e.g. no streaming, no key press), the scroll position changes internally but the viewport doesn't repaint — user sees a stale view until the next unrelated event. Fix: pass `immediate=True` to `scroll_relative()` / `scroll_to()` in any synchronous event handler that needs the scroll to be visible immediately. Applied to `OutputPanel.on_mouse_scroll_up` and `on_mouse_scroll_down` (2026-04-13).
- **`scroll_y` reactive has `repaint=False` — overriding `watch_scroll_y` without calling `_refresh_scroll()` silently breaks scroll repaint.** `Widget.scroll_y` is declared as `reactive(0.0, repaint=False, layout=False)`. Setting it does NOT auto-repaint. The base `Widget.watch_scroll_y(old, new)` calls `self._refresh_scroll()` (sets `_scroll_required=True` + `check_idle()`) when `round(old) != round(new)`. If you override `watch_scroll_y` without preserving this call, the viewport only repaints when an unrelated event (e.g. keypress) happens to trigger a refresh. **Fix:** always call `self._refresh_scroll()` inside any `watch_scroll_y` override when the value changes. Use 2-arg signature `(old_y, new_y)` to get both values.
- **`_show_status()` in cli.py must be TUI-guarded — after `_tui_startup_display()` sets `self.console = ChatConsole()`, every `console.print()` routes to the TUI output panel.** Any `console.print()` call that runs after the TUI starts (including in `show_banner()`) feeds through `_cprint()` → output queue → OutputPanel. In TUI mode, the Textual `StatusBar` widget handles model/status display; `_show_status()` output would appear as stray content above the input. Fix: `if _hermes_app is not None: return` at the top of `_show_status()`.
- **`get_css_variables()` only returns ThemeManager-managed vars, not all TCSS-declared vars.** `$ghost-text-color: #555555` declared in `hermes.tcss` will NOT appear in `get_css_variables()` output — that dict only shows Textual's semantic color palette + `COMPONENT_VAR_DEFAULTS`. The declaration is still valid for TCSS parsing (rules referencing `$ghost-text-color` work). This is the expected behavior; don't add TCSS-only vars to `COMPONENT_VAR_DEFAULTS` unless they need to be user-overridable at runtime.
- **Non-modal focus-trapping overlays must yield to `InterruptOverlay` (permission prompts).** `AnimConfigPanel.on_blur` uses `call_after_refresh(self.focus)` to trap focus while visible. Without an explicit check, this trap fires on every blur event — including blurs caused by `InterruptOverlay._activate()` grabbing focus. The result: the interrupt prompt gets focus for one tick, then AnimConfigPanel steals it back on the next tick. **Fix:** at the top of any `on_blur` refocus trap, check `if interrupt_overlay.has_class("--visible"): return`. Also requires `InterruptOverlay` to be on its own CSS layer (`layer: interrupt`) above `overlay`, so it renders on top visually. Screen layers: `base overlay interrupt tooltip`.
- **Rich markup in Button labels: `[reset]` renders empty.** Textual/Rich interprets `Button("[reset]")` as Rich markup — `[reset]` is a style-reset tag, so the label renders as empty string. Any `[word]` string passed to `Button()` is processed as markup. **Fix:** escape with `"\\[reset]"` (Python `"\\[reset]"` → string `\[reset]` → Rich renders as literal `[reset]`). Also affects `Static`, `Label`, any rich-markup-accepting widget. Always escape brackets in button labels unless you intend markup.
- **`Widget.__new__(Widget)` bypasses `Widget.__init__` — reactive storage broken.** `SubAgentPanel.__new__(SubAgentPanel)` leaves `_reactive_*` dict uninitialized → `ReactiveError: Node is missing data` on first reactive access. Always use proper constructors in tests: `SubAgentPanel(depth=0)`, not `object.__new__(SubAgentPanel)`.
- **`_pending_children` is a Textual Widget internal list attribute.** `Widget.__init__` sets `self._pending_children = []` as a private list. If you declare `self._pending_children: dict = {}` on a Widget subclass's `__init__`, it gets overwritten by `super().__init__()`. Symptom: `'list' object has no attribute 'setdefault'` at runtime. Use a different name like `_child_buffer`.
- **`border-left` in TCSS does not accept custom `$var-name` references.** `border-left: vkey $text-muted` fails silently or raises at parse time. Use literal hex colors: `border-left: vkey #666666 60%`. This is specific to `border-*` shorthand properties; color rules like `color: $text-muted` work fine.
- **TCSS rule order matters for equal-specificity class selectors.** `Widget.--collapsed` and `Widget.--has-children` both have specificity 0,1,1. The last-declared rule wins. If `--collapsed` should hide the body even when `--has-children` is set, declare `--collapsed` AFTER `--has-children` in the stylesheet.
- **`classify_tool("Task")` → `ToolCategory.UNKNOWN`, not `ToolCategory.AGENT`.** Only `"think"`, `"plan"`, `"delegate"` map to AGENT. In tests that need AGENT stack push, use `tool_name="delegate"` (not `"Task"`).
- **Patching a function imported inside another function: patch at source module.** If `message_panel.py` does `from hermes_cli.tui.tool_group import _maybe_start_group` inside the function body, patching `hermes_cli.tui.widgets.message_panel._maybe_start_group` won't work (name not in that module's namespace). Patch `hermes_cli.tui.tool_group._maybe_start_group` (source) instead — the `from X import Y` at call time fetches the patched version from the source module dict.
- **`Widget.size`, `Widget.app`, `Widget.content_size`, `Widget.is_mounted` are read-only properties (no setter).** `object.__new__(SomeWidget)` creates a bare instance where setting these raises `AttributeError: property 'X' of 'Y' has no setter`. The property is a data descriptor so even `obj.__dict__['size'] = ...` won't work (property overrides instance dict). In tests, use `patch.object(type(obj), 'size', new_callable=PropertyMock, return_value=Size(80,1))` as a context manager. For `app`, same pattern. For widgets that need reactives set, always call `Widget.__init__` via the normal constructor (e.g. `SubAgentPanel(depth=0)`) — `object.__new__` bypasses reactive storage setup.
- **`border_title` and `border_subtitle` are reactives — setting them on `object.__new__` instances raises `ReactiveError`.** They use Textual's reactive system which requires `Widget.__init__` to have been called. In tests that only need to verify these are set in `on_mount`, use `inspect.getsource(Cls.on_mount)` instead of actually calling `on_mount` on a raw object.
- **`call_after_refresh` flush can crash tests that pass MagicMock as Widget.** If `_mount_nonprose_block(mock, parent_tool_call_id="missing")` buffers the mock and schedules a flush, the flush runs after `await pilot.pause()` and tries `self.mount(mock)` → `AppError: Can't register MagicMock; expected a Widget instance`. Use real `Static("x")` widgets, or assert the buffer state synchronously before the pause.
- **`Static` has no `.renderable` attribute in Textual 8.x — use `.content`.** `static.renderable` raises `AttributeError`. The correct attribute is `static.content` (a `RenderableType`). `str(static.content)` gives the text. This applies to all `Static` instances and subclasses including `CopyableRichLog`-backed ones.
- **Patching a function-local import: patch at the source module, not the calling module.** `from agent.usage_pricing import estimate_usage_cost` inside a method body means the function fetches `estimate_usage_cost` from `sys.modules["agent.usage_pricing"]` at call time. `patch("hermes_cli.tui.overlays.estimate_usage_cost")` fails (name never bound there). Use `patch("agent.usage_pricing.estimate_usage_cost")` — patches the source module dict, which the local import then reads.
- **Duplicate `on_resize` methods in the same class — second silently shadows first.** Python allows redefining `def on_resize(self, event)` twice in one class body; the second definition wins silently. If you add a new `on_resize` (e.g. for D2 narrow-layout logic), grep `def on_resize` in the file first, then merge all logic into a single method.
- **`set_counts()` overwrites narrow button labels set by `_sync_narrow_layout()`.** `_sync_narrow_layout()` sets `down_btn.label = "[↓pg]"` on narrow resize, but `set_counts()` unconditionally sets `down_btn.label = f"[↓+{step_below}]"`. Result: narrowing the window then calling `set_counts()` reverts the narrow label. Fix: check `if self._narrow: down_btn.label = "[↓pg]" else: down_btn.label = f"[↓+{step}]"` inside `set_counts()`.
- **`Static` widget content is name-mangled to `_Static__content`.** `Static` stores its renderable as `self.__content` (double-underscore). Accessing it from outside the class requires the mangled form `widget._Static__content`. In tests, use this to assert the rich `Text` object a Static currently displays: `assert "▸" in widget._Static__content.plain`.
- **`ToolPanel.__init__` requires `block: Widget` as first positional arg.** `ToolPanel(tool_name="Bash")` raises `TypeError`. Correct: `ToolPanel(block=Static("body"), tool_name="Bash")`.
- **`Widget.remove()` is async — `is_mounted` stays `True` after a single `await pilot.pause()`.** After calling `child.remove()` and `await pilot.pause()`, `child.is_mounted` can still be `True` because the removal is scheduled. Use `await pilot.pause(delay=0.1)` or test the observable side-effect (CSS class state, parent children count) instead of checking `is_mounted` directly.
- **`Chip` dataclass requires `kind` before `tone`.** `Chip(text="x", tone="ok")` raises `TypeError: missing 1 required positional argument: 'kind'`. Correct: `Chip(text="x", kind="exit", tone="ok")`.
- **`ToolPanel.--minified` class vs `display = False` for mini-mode.** Setting `panel.display = False` kills layout space entirely and can cause edge cases in parent height calculation. Use `panel.add_class("--minified")` (CSS: `height: 0; overflow: hidden;`) to hide the panel while preserving its DOM slot — especially important when `ToolPanelMini` sits adjacent and needs the slot's position.
- **`? → show_help`, `F1 → show_context_menu` (vim keybind convention).** Previous passes accidentally reversed these. Canonical mapping: `question_mark` → `show_help` (vim ?, opens inline help overlay); `f1` → `show_context_menu` (function key, opens context/actions menu). Tests checking `ToolPanel.BINDINGS` should assert both.
- **`OmissionBar` force-show when `cap_msg` present (D3 pattern).** When all lines are visible (`visible_end == total`) but truncated-line warnings exist, `cap_msg` is truthy. Use `show_bottom = (visible_end < total) or bool(cap_msg)` so the bar remains visible and shows the truncation warning even when no lines are hidden.
- **Method-body attribute assignment resets on every call.** `self._foo: list = []` inside a method body is a valid Python attribute assignment — it just also fires every time the method is called. `ResponseFlowEngine._code_fence_buffer = []` placed at the end of `_write_prose()` reset the accumulation buffer on every prose write, making InlineCodeFence detection unreachable. Always initialize instance attrs in `__init__`, never in method bodies.
- **Function-local imports are NOT shared between methods.** `from agent.rich_output import apply_block_line` inside `process_line()` binds that name in `process_line`'s local scope only. `flush()` calling `apply_block_line` without its own import raises `NameError`. Each method that needs a lazily-imported name must import it locally, OR import at module level.
- **`StreamingBlockBuffer` holds one line in lookahead — numbered lines arrive one step late.** SBB buffers one line for setext/table detection; it returns the PREVIOUS line when given the NEXT. In `process_line("line2")`, the SBB returns `line1`. Consequence: a pre-flush before `_commit_prose_line` fires between every two numbered lines, preventing accumulation. Fix: place the buffer flush only in the emoji-else path; let `_commit_prose_line` handle flushing when a non-numbered line arrives.
- **All prose-writing paths must funnel through the same accumulator.** When `_commit_prose_line` was wired to accumulate numbered lines, `_flush_block_buf()` still wrote directly to `_prose_log` — lines flushed at turn-end escaped the accumulator. Any InlineCodeFence whose last numbered line was the final SBB-buffered line never mounted. Fix: replace the direct `_prose_log.write_with_source` in `_flush_block_buf` with `_commit_prose_line`.
- **`_try_render_mermaid_async()` must use `self.app.call_from_thread()`, not `self.call_from_thread()`.** `StreamingCodeBlock` is a Widget, not App — `Widget.call_from_thread` does not exist in Textual 8.x. Always call `self.app.call_from_thread(fn, *args)` from daemon threads spawned from widget methods.

- **`density-compact` + `border: tall` on `HermesInput:focus` collapses content area to 0.** `max-height: 2` in compact mode + `border: tall` (1 top + 1 bottom = 2 rows) leaves content_size.height=0. Cursor invisible; input appears unfocusable. Fix: `HermesApp.density-compact HermesInput:focus { border: none; }`. Test: `assert inp.content_size.height > 0` after focusing in a compact-sized pilot.
- **PlanPanel must declare `display: none` in DEFAULT_CSS.** Without it, the panel renders empty sections at startup (5-6 rows) and displaces the input row. Add `display: none` to the base rule and `PlanPanel.--active { display: block; }` for the active state. Control via `_on_planned_calls_changed`: `self.add_class("--active")` when `bool(calls)`, remove otherwise. hermes.tcss reorganizations can silently drop rules — always verify DEFAULT_CSS after tcss refactors.
- **`ReasoningPanel.close_box` order: hide `_live_line` BEFORE writing to `_reasoning_log`.** The original order (write to log → hide `_live_line`) left a Textual render window where both widgets showed the same last-line content, producing visible duplication. Correct order: capture `buf`, clear `_live_buf`, hide `_live_line`, call `_live_line.update("")`, then call `process_line(buf)`. This eliminates the overlap regardless of when Textual batches its render.
- **Scrolling container reduces content width by 1 — use `scrollable_content_region.width` for render width.** `OutputPanel` has `scrollbar-size-vertical: 1`. When the scrollbar is visible, `op.scrollable_content_region.width` = `op.size.width - 1`. Sibling widgets (e.g. `TitledRule`, `PlainRule`) that render at `self.size.width` will overflow by 1 column into the scrollbar gutter. Fix: query the scroll container and clamp: `w = min(self.size.width, op.scrollable_content_region.width)`. Use `try/except Exception: pass` so the rule still renders standalone (tests, no OutputPanel).
- **Rich `Color.__str__` normalises hex to lowercase; use `.lower() in str(...).lower()` for comparisons.** `SkinColors.default().error = "#E06C75"` → stored as `"#e06c75"` in Rich spans. `"#E06C75" in str(span.style.color)` fails. Always compare with `.lower()` on both sides.
- **`rich.console.Group` doesn't stringify to rendered content — access `._renderables`.** `str(Group(...))` returns Python object repr. In tests, iterate `group._renderables` and extract `.plain` from each `Text` member.
- **`Rich.Text.append()` has no `end=` kwarg.** `text.append("x", end="")` → `TypeError`. Omit it — append always concatenates.
- **Float field truthiness: `0.0` is falsy — use `is not None`.** `if finished_at and started_at` skips when `started_at=0.0`. Always `if finished_at is not None and started_at is not None` for timestamp/numeric fields that can be zero.
- **Deleting `@work`-decorated DEPRECATED forwarders: scan `overlays/` and `services/` for `app._method_name(...)` callers.** `grep self._method` only finds callers inside the class itself; external callers in service/overlay files use `app._method_name(...)` and are missed. When the deleted method was `@work`-decorated, replace callers with `app.run_worker(app._svc_foo.actual_method(arg), thread=False)`. Found during D7 cleanup: `_adapters.py` and `services/keys.py` called `app._run_undo_sequence` / `app._run_rollback_sequence` which spec had listed as zero-caller.
- **`Widget.watch(obj, attr, cb)` returns `None` in Textual 8.2.3 — never store or stop the handle.** The method signature is `-> None`. Storing `self._handle = self.watch(...)` and then calling `self._handle.stop()` in `on_unmount` will raise `AttributeError: 'NoneType' object has no attribute 'stop'` on every app shutdown. Textual automatically unregisters cross-widget watchers when the observing widget is unmounted — no manual cleanup is required. Correct pattern: `self.watch(app, "my_reactive", self._on_change)` in `on_mount`; `on_unmount` only needs to stop any timers/animations the widget owns directly (e.g. `self._pulse_stop()`, `self._shimmer_stop()`).
