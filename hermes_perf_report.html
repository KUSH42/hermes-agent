Methodology & Coverage

✅ Median over 200–2000 iterations — resistant to OS scheduler noise
✅ 60fps frame budget (16.67ms) as primary rendering threshold
✅ All critical hot paths covered: streaming, autocomplete, event loop
✅ Headless Textual — deterministic layout, reproducible in CI
⚠️ Headless adds ~10–15ms fixed overhead per frame — widget numbers are conservative
⚠️ Micro-benchmarks only — no end-to-end user scenario measurement
⚠️ Single machine / single point in time — no regression history
⚠️ Path walker test requires real filesystem I/O (skipped in offline CI)
Real-time safety claim: A terminal UI is "real-time safe" when every hot-path operation completes within the 16.67ms frame budget (60fps), ensuring no user-visible input lag or rendering jank. Hermes enforces this via: (1) a bounded async queue (4096 entries) decoupling agent output from the event loop, (2) O(viewport) virtual scroll for the completion list regardless of candidate count, (3) exclusive background workers preventing unbounded task accumulation, and (4) timer delivery tests proving the event loop is never starved during peak streaming load.

CI enforcement: set PERF_STRICT=1 before running pytest -m slow tests/tui/test_streaming_perf.py tests/tui/test_autocomplete_perf.py to promote all timing warnings to hard assertion failures.
