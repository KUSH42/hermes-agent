# Real-Terminal tmux Audit Driver

Companion to `live-audit.md`. The Pilot harness in that doc runs headless — no real PTY, no kitty/sixel probes, no real SIGWINCH. Some bugs only surface through a real terminal. This file is the procedure for running one slow "soak" pass per audit cycle through tmux.

> When to use: **after** the Pilot pass produces its findings list, **at most once per cycle**, **only for flows that exercise TTY-side behavior**. Never as a replacement for Pilot — Pilot is 10–40× faster and has structural assertions tmux can't replicate.

Spec of record: `~/.hermes/2026-04-28-tmux-audit-driver-spec.md`.

---

## 1. When tmux earns its cost

Pilot's screen buffer is synthetic — it parses ANSI into a virtual grid in-process. That misses:

- **Real `os.isatty()`** — kitty_graphics, sixel/APC probes early-return under headless. Real probes hit the terminal and parse responses.
- **Real SIGWINCH** — `pilot.resize_terminal()` synthesizes the event in Textual's event loop. tmux fires the actual signal; resize-handler races and reflow ordering bugs only show under real signal delivery.
- **Real ANSI byte stream** — render bugs that only appear when sequences interleave at PTY speed (cursor-positioning races, partial-frame tearing) are invisible to Pilot.
- **Real OSC color queries** — `_refresh_colors` under Pilot reads Textual's defaults (e.g. the `auto 87%` round-2 leak). A real terminal answers OSC 11 / OSC 4 with the user's actual palette; bugs in palette consumption only surface here.
- **Real focus events** — `\e[I` / `\e[O` for focus-in/out aren't generated by Pilot.
- **Real scrollback overflow** — Pilot's buffer is bounded and tidy. tmux's scroll region exposes scroll-clipping and overdraw bugs.

If a finding's repro doesn't depend on any of these, **don't use tmux for it**. Pilot is cheaper.

## 2. Pre-flight

```bash
tmux -V                       # need ≥ 3.0 for resize-window flag syntax
which hermes                  # the binary the audit will drive
tmux ls 2>/dev/null           # check no stale audit sessions
```

If `tmux ls` shows leftover `hermes-audit-*` sessions from a crashed prior run, kill them: `tmux kill-session -t hermes-audit-xxx`. Periodic safe nuke: `tmux kill-server` (kills all your tmux sessions — only use if you have no other tmux work in flight).

## 3. Driver

`tools/tui_audit/tmux_driver.py` exposes a `TmuxDriver` context manager. Always use it in a `with` block — stale sessions accumulate fast.

```python
from tools.tui_audit.tmux_driver import TmuxDriver

env = {
    "HERMES_KEYSTROKE_LOG": "1",
    # Do NOT set HERMES_CI=1 — it suppresses the keystroke recorder.
    "TERM": "xterm-kitty",  # or xterm-256color depending on what you want to probe
}

with TmuxDriver("hermes", cols=200, rows=60, env=env) as drv:
    drv.wait_for(lambda s: "hermes>" in s, timeout=5)
    drv.send_keys("hello world", literal=True)
    drv.send_keys("Enter")
    drv.wait_for(lambda s: "Streaming" in s, timeout=10)
    print(drv.capture())
```

Key methods:

- `send_keys(keys, literal=False)` — `literal=True` for typed text (passes `-l` to tmux); leave default for named keys (`Enter`, `C-c`, `Up`, etc.).
- `capture()` — returns the full visible pane as a string. **No trailing whitespace stripping** — keep the raw output for cell-position assertions.
- `resize(cols, rows)` — fires real SIGWINCH. Use 200×60 → 80×24 → 200×60 to exercise reflow.
- `wait_for(pred, timeout, interval)` — polls `capture()`. Returns `False` on timeout (no exception); the audit decides what to do.

## 4. Flows worth a tmux pass

Cap at ≤ 3 flows / ≤ 5 minutes wall-clock per audit cycle. If you need more, split.

| Flow | What it catches |
|------|----------------|
| **Cold mount in real terminal** | kitty/sixel/APC probe paths; mount-budget under real `os.isatty()`; OSC color reads |
| **Resize sweep** (200×60 → 80×24 → 200×60, 1 s settle each) | SIGWINCH-handler races; reflow ordering; scrollback clipping; ToolPanel/MessagePanel layout under real signal delivery |
| **Long streaming overflow** | Scroll-region bugs; partial-frame tearing; cursor reset on scroll; live-line widget behavior past the visible region |
| **Focus loss/regain** (send `\e[O` then `\e[I` raw) | Focus-event handlers; nameplate idle vs active; thinking-widget animation gating |

Skip:

- Tool-call lifecycle assertions (Pilot's structural query is better).
- Density-tier toggle (Pilot's `pilot.app.query_one(ToolPanel)._resolver` introspection beats cell parsing).
- Skin/CSS-variable correctness via widget tree (Pilot wins; only use tmux if testing OSC palette ingestion).

## 5. Divergence triage

Run the same flow under both drivers — Pilot first (cheap), then tmux (expensive). Compare findings:

| Pilot pass | tmux pass | Verdict |
|------------|-----------|---------|
| Clean | Clean | Flow is healthy at both layers |
| Finding | Clean | Pilot-synthetic-state bug — investigate Pilot harness or Textual mock surfaces |
| Clean | Finding | TTY/render bug — only manifests through real PTY (e.g. probe paths, signal handlers, real ANSI) |
| Finding | Finding | Real logic-layer bug — both drivers agree |

Record divergences in the findings file under a dedicated `Pilot/tmux divergence` section. These are the highest-signal findings the cycle produces — they're what motivated adding the tmux pass in the first place.

## 6. Keystroke JSONL parity

The keystroke recorder (KL-1..KL-7, commit `db31b6c29`) hooks `App.on_key` / `App.on_mouse` regardless of input source. Both Pilot and tmux feed those hooks, so both produce JSONL.

**Parity check**: run the same flow against each driver, diff the JSONL files. They should match modulo timing (the JSONL stamps `event_ts_ms` per event). Structural divergence (different keys logged) is a recorder bug.

**Caveat**: KL-7 is partial — not every in-widget event has a hook. A flow that produces matching JSONL on both drivers can still hide a deep-component bug. Don't claim "tmux validates Pilot." The parity check is a regression net, not a proof.

---

## 7. Anti-patterns

- **Driving tmux for cheap unit-style assertions.** If you're not exercising a TTY-only behavior from §1, you're paying 50–200 ms/step for nothing.
- **Cell-grid assertions for widget state.** "The string `\"running\"` is at row 14" is brittle — width changes, theme changes, and font choice break it. Use Pilot for structural state; use tmux for "did this render at all" assertions.
- **Forgetting `with` / `__exit__`.** Stale sessions pile up. Always context-manage.
- **`HERMES_CI=1` in the env.** Disables the keystroke recorder. Audit gets no JSONL. Default-on dev environments may have this set — explicitly drop it.
- **One tmux pass per finding.** Wastes time. Pilot first, then one consolidated tmux pass.
- **Comparing tmux output across terminal types.** A run with `TERM=xterm-kitty` and one with `TERM=xterm-256color` will diverge legitimately. Pin `TERM` per audit cycle and document it in the findings header.

## 8. Cleanup

After every audit:

```bash
tmux ls 2>/dev/null | grep hermes-audit
# if anything shows up, the driver leaked — kill it and file a driver bug
```

Add the cleanup check to the audit's findings report under "Harness hygiene" — leaks are themselves findings about the audit tooling.
